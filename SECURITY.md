# Security Policy

## Supported Release

The latest tagged release is the supported release line.

## Reporting A Vulnerability

Do not publish sensitive details in a public issue. Use a private GitHub security advisory at:

https://github.com/MaiDuy708/system-audit-review/security/advisories/new

Include the affected file, a minimal reproduction, impact, and any safe mitigation. Reports are acknowledged after triage; no fixed response-time promise is made.

## Scope

Report issues that could make the skill induce unsafe mutation, expose credentials, misrepresent evidence, bypass explicit authorization, or compromise a supported installation path.
